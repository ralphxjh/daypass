# DayPass · 税务居住期计算器

一款专为跨国穿梭的商务人士、数字游民和外派员工设计的纯本地、零成本 PWA 工具。帮助你精准追踪每年在不同国家的停留天数，直观预警税务居民身份风险。

🌐 在线体验 Demo | 📱 手机浏览器打开可添加至主屏幕，体验原生 APP 效果

## ✨ 核心功能

- **精准日期计算**：支持跨年行程自动拆分，严格按照单自然年（1月1日-12月31日）计算实际停留天数。支持同日双算（入境日与出境日同天计为2天）。
- **护照印章式状态卡**：每个国家是一页"护照"，停留天数像盖章一样累积，状态做成歪斜的朱红入境章——SAFE 安全 / WATCH 警告 / ALERT 危险，无需看数字即可一眼判断风险。卡片底部还有仿护照机读码（MRZ）行。
- **自定义国家与阈值**：内置中/日/新三国模板，支持自由添加任意国家，并自定义税务居民判定天数阈值。
- **"仍在境内"支持**：处理未出境的现实场景，系统自动以"今天"作为出境日期进行动态计算。
- **中英双语界面**：顶部标题栏直接切换，中文/英文全部 UI 文字即时转换。语言偏好自动记忆。
- **时间重叠检测**：添加出入境记录时自动检测与已有记录的时间冲突（同一天交接允许，多天重叠禁止），弹窗内红色横幅提示。
- **存储可用性检测**：首次加载自动检测浏览器 localStorage 是否可用，不可用时页面顶部显示红色警告。
- **数据完全本地化**：所有数据仅存储在浏览器 localStorage 中，绝不上云，保护敏感行程隐私。
- **一键导入/导出**：支持将数据导出为 JSON 文件备份，支持从文件恢复（已完美兼容 iOS/安卓文件读取机制），换机无忧。
- **PWA 离线可用**：安装至主屏幕后，即使断网也能正常打开和使用。

整体采用"护照"视觉语言，与"跨境停留"主题高度契合：

### 核心计算逻辑 (`calculator.js`)

采用 effectiveStart 和 effectiveEnd 截断算法，将所有行程与目标年份求交集，确保跨年行程不会误算到非当年天数。计算公式为：天数 = 交集结束日 - 交集开始日 + 1（同日双算）。

### 国际化架构

不使用任何第三方 i18n 库。翻译数据集中在 `src/i18n/translations.js`，通过 React Context 注入全局。`t('path.to.key', { vars })` 函数按点号路径查找翻译并替换 `{{变量}}` 占位符。将来添加新语言只需在 `translations.js` 中新增语言对象，并在 NavBar 的 `<select>` 中加一个 `<option>`。

## 📱 如何安装到手机桌面（PWA）

本应用支持作为 PWA 安装到手机主屏幕，获得类似原生 APP 的全屏体验：

- **iOS (Safari)**：点击底部分享图标 ➔ 向下滑动找到并点击"添加到主屏幕"。
- **Android (Chrome)**：浏览器会自动弹出"添加到主屏幕"底栏，或点击右上角菜单手动添加。

⚠️ 必须通过 HTTPS 链接访问时，安装功能才会生效。

## ⚠️ 免责声明

**仅供参考，不构成税务建议**：本工具仅基于单自然年物理停留天数进行数学计算，税务居民的判定涉及诸多复杂法律因素（如住所、习惯性居所、高管身份等），请以当地税法及专业税务师意见为准。

**单一年份模型**：目前系统采用"单自然年"计算模型，完美适配中、新等国的 183 天规则。但对于如美国的实质居住测试（当年+前两年加权）等滚动周期规则，本工具无法自动适配，需用户自行评估。

**浏览器隐私模式**：在无痕/隐私模式下使用，localStorage 可能会在关闭窗口后被清空，且 PWA 离线功能受限，请使用常规浏览器模式。
